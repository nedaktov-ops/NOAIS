# v0.6 — Facebook Adapter Design

**Date:** 2026-06-07
**Branch:** `feature/v0.6-facebook`
**Target:** v0.6.0 release

## Goal

NOAIS should detect AI-likely Facebook posts and comments and visually mark them — using the same adapter pattern as v0.5 YouTube.

## Scope (in)

- Detect Facebook post elements (`[role="article"]` for posts; `[aria-label*="Comment"]` or `div[class*="Comment"]` for comments).
- Run heuristics + phrase counter on each.
- Append a NOAIS badge with a colour-coded score.
- Soft + Hard mode (same as YouTube).
- MutationObserver for infinite scroll.
- 100% on-device, no network, no innerHTML.

## Scope (out)

- Detecting AI-generated images or videos on FB.
- Detecting AI in Messenger (separate DOM, separate spec).
- Hard-mode options UI (still deferred to v0.5.1 across all adapters).
- Reply threads (top-level only).

## Architecture (no changes from v0.5)

Reuse everything from v0.5:
- `core/adapters/base.js` (interface + helpers)
- `core/adapters/youtube.js` (sibling)
- `styles/adapters.css` (shared)
- `content/content.js` (adapter dispatch by hostname)

Add: `core/adapters/facebook.js` — implements the BaseAdapter contract for FB.

### Facebook adapter

| Aspect | Decision |
|---|---|
| Hostname match | `facebook.com`, `m.facebook.com`, `fb.com`, `fb.me` |
| Post selector | `[role="article"]` (works in 2024+ FB UI; covers both posts and comments in the news feed). |
| Text extraction | First `div[dir="auto"]` inside the article that has text length >= 30 chars (skip "Like", "Reply" buttons, etc.). Fall back to `article.textContent`. |
| Decorate | Same as YouTubeAdapter: `noais-score-{severity}` class + appended badge via createElement+textContent |
| shortTextMode | true (FB posts are usually short, like YouTube comments) |
| Soft / Hard mode | Same as YouTube. Read from `noais_hard_mode_sites[host]`. |
| Skip conditions | Text under 30 chars or fewer than 5 words (delegated to `helpers.shouldScore`). |

### Facebook's mutable DOM is hostile

FB heavily rewrites the DOM. **Each scan is best-effort**; we may double-decorate if a comment is removed and re-added. Mitigation:
- Decorate is idempotent (checks `dataset.noaisScanned`).
- MutationObserver fires rAF-throttled.
- We never remove existing decorations, so a re-decorated comment keeps its badge.

## Storage schema (additive, no migration)

```json
{
  "noais_enabled": true,
  "noais_global_sensitivity": 100,
  "noais_site_overrides": {},
  "noais_hard_mode_sites": {}  // unchanged from v0.5
}
```

The `noais_hard_mode_sites` key already supports any host. v0.5 shipped with YouTube being the only adapter that consulted it; v0.6 adds FB.
