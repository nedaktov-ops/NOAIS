# v1.0 Design — Public Release

**Date:** 2026-06-07
**Author:** NedCode3 + vibe-coder
**Status:** Approved (proceeding)

## 1. Goal

Ship NOAIS v1.0 to the public. Make it trivial for a stranger to:

1. **Download** a pre-built `.zip` from GitHub Releases.
2. **Install** it in Chrome or Firefox (or via Firefox AMO).
3. **Understand** what it does, what it doesn't do, and what data it touches (privacy).

The "1.0" milestone is a packaging + documentation milestone, not a code milestone. The four platform adapters (YouTube, Facebook, Instagram, TikTok), 199-test suite, CI on GitHub Actions, and stable extension ID are all already in place.

## 2. Non-goals

- No new features. v1.0 is "what we have, packaged well."
- No Chrome Web Store submission. That requires a $5 one-time developer fee and a different submission process; not in v1.0 scope.
- No "Pro" tier. NOAIS is Apache-2.0 and stays that way.
- No auto-update infrastructure. v1.0 ships a `.zip`; users re-download for new versions.
- No signing infrastructure for the Chrome build. Chromium will warn on unpacked install — that's expected and fine for v1.0 (Firefox users get a clean install via AMO).

## 4. The `make package` target

```makefile
package:
	@mkdir -p dist
	@rm -rf dist/NOAIS-v1.0.0-*.zip
	@cd extension && zip -r ../dist/NOAIS-v1.0.0-chrome.zip . -x '*.DS_Store'
	@cp dist/NOAIS-v1.0.0-chrome.zip dist/NOAIS-v1.0.0-firefox.zip
	@echo "Built:"
	@ls -lh dist/
```

Both zips are identical (Chrome and Firefox both accept a standard zip; the manifest is MV3 with both `manifest_version: 3` and `browser_specific_settings.gecko` so it works in both browsers unchanged).

## 5. PRIVACY.md

This is the **most user-facing legal document** in the project. It must be:

1. **Plain language.** No "we may collect anonymized telemetry" handwaving.
2. **Honest.** "We do not collect, transmit, or store any data on any server, ever."
3. **Specific.** List exactly what data lives in the user's browser, and where.
4. **AMO-compliant.** Firefox's add-on review requires a privacy policy that discloses whether the extension handles personal data, even if the answer is "no".

Draft outline:

```markdown
# NOAIS Privacy Policy

NOAIS is a fully on-device browser extension. **It does not collect, transmit, or store
any data on any server.** It does not make network requests of any kind. The only data
that exists is stored locally in your browser via the standard `chrome.storage` API
(and Firefox's equivalent `browser.storage`).

## What data lives in your browser?

| Key | Purpose | Lifetime |
|---|---|---|
| `noais_enabled` | Whether the extension is on. | Until you uninstall. |
| `noais_global_sensitivity` | 0–100 slider. | Until you uninstall. |
| `noais_site_overrides` | Per-site enable/disable. | Until you uninstall. |
| `noais_hard_mode_sites` | Per-site hard-mode toggle. | Until you uninstall. |

All four are **local-only**. They never leave the browser. They are not synced, not backed
up, not uploaded, not used for analytics, and not shared with any third party.

## What does the extension scan?

The content script reads the text of the page you are viewing, in-memory only, to
score it for AI-likely patterns (heuristics). The text is never written to disk, never
sent over the network, and never leaves the page's JavaScript context.

## Permissions

- `storage` — to read/write the four keys above.
- `activeTab` — to access the current page's URL (so the extension can decide whether
  to run on a whitelisted site).
- `<all_urls>` (host permission) — required because NOAIS is opt-in per-site, and the
  per-site list is user-extensible. NOAIS does not send the URL anywhere.

## Open source

NOAIS is Apache-2.0 licensed. Source code: https://github.com/nedaktov-ops/NOAIS
You can audit every line.
```

## 6. EXTENSION-LISTING.md (Firefox AMO submission prep)

This file is the **submission metadata** the user pastes into addons.mozilla.org when submitting. It's pre-formatted so the user just copies the fields.

```markdown
# NOAIS — Firefox AMO Listing

## Name
NOAIS — No Artificial Intelligence Slop

## Summary (≤ 250 chars)
Detects and masks AI-generated text on YouTube, Facebook, Instagram, TikTok. 100% on-device. No accounts. No telemetry. Apache-2.0.

## Description (long)

NOAIS is a free, open-source browser extension that helps you detect, mask, and disable
AI-generated content on the web.

- **Detects** AI-likely text on YouTube comments, Facebook posts, Instagram captions, and
  TikTok comments using on-device stylometric heuristics.
- **Decorates** each scored element with a small badge showing the score (0–100) and a
  colour-coded severity outline.
- **Masks** suspected AI content in "hard mode" (dim + blur) — toggleable per site.
- **Respects your privacy**: 100% on-device, no network calls, no accounts, no telemetry.
- **Customisable**: per-site enable/disable, global sensitivity slider.
- **Open source**: Apache-2.0.

## Categories
- Privacy & Security
- Productivity
- Social & Communication

## Tags
ai, artificial intelligence, gpt, chatgpt, slop, spam, filter, mask

## License

## Privacy policy
https://github.com/nedaktov-ops/NOAIS/blob/main/PRIVACY.md

- `activeTab`
```

## 8. Acceptance criteria

- `make package` produces two identical `dist/NOAIS-v1.0.0-*.zip` files.
- `releases/v1.0.0.md`, `PRIVACY.md`, `EXTENSION-LISTING.md` exist and are well-formed.
- CHANGELOG has a v1.0.0 entry.
- README "Install" section is filled in (no more "coming in v1.0").
- Tag `v1.0` is pushed to remote.
- GitHub Release draft is created with the two zips attached.
- 199/199 tests still green.
- Local backup `NOAIS-v1.0-<ts>` exists.

## 9. What the user must do manually (printed at end of session)

- Review the PRIVACY.md wording (it's the user-facing legal doc).
- Submit the Firefox AMO listing using EXTENSION-LISTING.md (requires an addons.mozilla.org account; review takes 1–7 days).
- Optional: Chrome Web Store submission ($5 dev fee).
- Optional: promote the GitHub Release on social media / Reddit / HN.

## 10. Out of scope (deferred)

- **Chrome Web Store.** Requires $5 dev fee + Google account. Can be done anytime; just upload the existing `dist/NOAIS-v1.0.0-chrome.zip`.
- **Auto-update.** NOAIS has no update server. Users re-download from GitHub Releases.
- **Telemetry / analytics.** Will never be added (Apache-2.0 + privacy promise).
- **Localisation.** v1.0 ships in English only. v1.1+ can add i18n if there's demand.
