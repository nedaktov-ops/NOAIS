# v0.5 — YouTube Adapter Design

**Date:** 2026-06-07
**Branch:** `feature/v0.5-youtube`
**Target:** v0.5.0 release

## Goal

NOAIS should detect AI-likely comments on YouTube and visually mark them — without breaking the page or the user's reading flow.

## Scope (in)

- Detect YouTube comment elements (`ytd-comment-renderer`).
- Run heuristics + phrase counter on each comment's visible text.
- Append a small "NOAIS" badge with a score, colour-coded by severity.
- Style the comment container with a CSS class for masking (Soft + Hard modes).
- Watch for new comments via `MutationObserver` (infinite scroll).
- 100% on-device, no network calls.

## Scope (out)

- Detection of AI-generated videos (v0.5 is **comments only**).
- Detecting AI-generated thumbnails or titles.
- Auto-hiding comments (Soft mode = badge only; Hard mode = visual dim, NOT a hide — keeps user's control).
- Reply threads (only top-level comments in v0.5; v0.5.x can extend).

## Architecture

### Adapter pattern

```
extension/
├── core/
│   ├── adapters/
│   │   ├── base.js          (BaseAdapter interface)
│   │   └── youtube.js       (YouTube-specific: selectors, badge creation)
│   ├── heuristics.js        (existing)
│   └── settings.js          (existing)
├── content/
│   └── content.js           (refactored: pick adapter by hostname, run it)
└── styles/
    └── adapters.css         (shared CSS for badges + masking)
```

### Adapter contract (`core/adapters/base.js`)

```js
// All adapters expose this shape. They run in the content-script world.
class BaseAdapter {
  static id = 'base';                // string, used in logs + CSS classes
  static match(hostname) { ... }     // returns true if this adapter applies
  findElements(root) { ... }         // returns NodeList of comment-like elements
  extractText(element) { ... }       // returns the visible text of one element
  decorate(element, score, count)    // injects badge + CSS class, no innerHTML
}
```

### Content script integration

```js
// content/content.js (sketch)
const HOST = settings.parseHostname(location.href);
const ADAPTERS = [YouTubeAdapter /*, FacebookAdapter, ... */];
const adapter = ADAPTERS.find(a => a.match(HOST));
if (!adapter) { /* fall back to phrase counter only */ }
const root = document.body;
const observer = new MutationObserver(scan);
observer.observe(root, { childList: true, subtree: true });
scan();
function scan() {
  const elements = adapter.findElements(root);
  for (const el of elements) {
    if (el.dataset.noaisScored) continue;
    el.dataset.noaisScored = '1';
    const text = adapter.extractText(el);
    const { score, count } = scoreText(text, effective);
    adapter.decorate(el, score, count);
  }
}
```

### YouTube adapter (`core/adapters/youtube.js`)

| Aspect | Decision |
|---|---|
| Hostname match | `youtube.com`, `m.youtube.com`, `youtu.be` (the last redirects to YT) |
| Selector | `ytd-comment-renderer` (works in 2024+ UI). Fallback: `#content-text` direct children of `ytd-comment-thread-renderer`. |
| Text extraction | `element.querySelector('#content-text')?.textContent ?? ''` |
| Decorate | `element.classList.add('noais-score-' + severity)`; append `<span class="noais-badge">NOAIS 81</span>` via `createElement` + `textContent` (no innerHTML) |
| Hard mode | Adds `.noais-hard { opacity: 0.35; filter: blur(2px); }` on the element |
| Soft mode (default) | Adds badge only, leaves comment readable |
| Skip conditions | If `text.length < 30` chars: skip score, skip badge (avoids noise on "ok", "lol" comments) |

### Hard vs Soft mode

- **Soft** (default in v0.5): badge appears, comment stays readable. The user can decide what to do.
- **Hard** (toggleable per-site, defaults to OFF): comment gets visually dimmed + blurred. **Never hidden** — the user can still click through.
- Mode is read from `chrome.storage.local.get('noais_hard_mode_sites', ...)` — defaults to `{}`. User toggles via options page (deferred to v0.5.1, v0.5 ships with badge-only).

### Heuristics on short text

The existing heuristics engine returns `score: 0` for texts under 50 words. v0.5 adds a new option to the engine: `analyzeText(text, { shortTextMode: true })` which lowers the minimum to 5 words and uses a simpler algorithm (TTR + entropy only, no burstiness). This is **opt-in only** — defaults unchanged.

### Visual design

Badge (Soft mode):

```
┌─────────────────────────────────────────────────┐
│ UserName • 2 days ago                           │
│ ┌─[ NOAIS 81 ]─┐                                │
│ │ This is an AI-generated comment that...      │
│ │ continues with another AI-typical sentence... │
│ └──────────────┘                                │
└─────────────────────────────────────────────────┘
```

- Badge: small, inline, `font-size: 0.7em`, `padding: 1px 6px`, `border-radius: 3px`.
- Colours: green `<=30`, amber `31-60`, red `>=61`. Light + dark mode aware.
- Hard mode adds a `filter: blur(2px) opacity(0.35)` to the entire comment text container, plus a hover-reveal (`filter: none`) so the user can still read on demand.

## Storage schema (additive)

```json
{
  "noais_enabled": true,
  "noais_global_sensitivity": 100,
  "noais_site_overrides": { ... },
  "noais_hard_mode_sites": { }       // NEW: defaults to {} (soft mode everywhere)
}
```

No migration needed.
